# Architect Training OS

## 从语法到系统架构师的工程成长路线

> Build Systems. Break Systems. Fix Systems. Become an Architect.

本项目旨在打造一套**工程化、可执行、可递进的架构师成长训练系统**，通过真实基础设施项目驱动学习，而非碎片化技术学习。

目标不是"学更多技术"，而是：

> **获得驾驭复杂系统的能力。**

------------------------------------------------------------------------

# 一、核心设计理念

## ✅ Project-driven Learning

所有技术必须通过真实系统实现掌握，而非停留在理论或Demo。

## ✅ 单语言主线

-   核心语言：**Golang**
-   所有基础设施优先使用 Go 实现
-   避免多语言认知切换

## ✅ Learning as Code

提供可维护的 roadmap： - 支持增删改 - 可扩展 - 可标记完成状态 -
与项目强关联

------------------------------------------------------------------------

# 二、架构成长的三次跃迁

## ⭐ 第一次跃迁：组件工程师 → 系统工程师

标志：开始构建基础设施，而不是业务系统。

**项目：** - mini-mysql - mini-redis - mini-mq

**能力提升：** - IO模型理解 - 并发控制 - 数据结构工程化 - 性能分析

------------------------------------------------------------------------

## ⭐ 第二次跃迁：系统工程师 → 分布式工程师

标志：能处理多节点问题。

**项目：** - RPC Framework - Distributed Cache - Raft / etcd-like

**挑战：** - 网络不可靠 - 时钟漂移 - Partial failure - Split brain

------------------------------------------------------------------------

## ⭐ 第三次跃迁：分布式工程师 → 架构师

关注： - 数据流 - 流量治理 - 故障域 - 成本控制

**项目：** - API Gateway - Observability Platform - Chaos Platform

------------------------------------------------------------------------

# 三、中间件学习顺序（架构师版本）

    Storage → Cache → Queue → RPC → Coordination → Gateway

顺序：

1.  MySQL\
2.  Redis\
3.  Kafka\
4.  RPC\
5.  etcd / ZooKeeper\
6.  Gateway

**原因：** \> 架构本质 = 数据 + 流量

------------------------------------------------------------------------

# 四、中间件五层掌控模型

## 1️⃣ Production Usage

像生产环境一样部署： - Docker Cluster - 压测（wrk / k6 / sysbench）

必须知道系统极限。

------------------------------------------------------------------------

## 2️⃣ Architecture & Internals

只关注：

-   数据如何存？
-   请求如何流动？
-   为什么性能高？

避免源码崇拜。

------------------------------------------------------------------------

## 3️⃣ Failure & Debugging

固定排障路径：

1.  看CPU / IO / Load\
2.  查连接\
3.  查慢查询 / 热Key\
4.  火焰图分析

------------------------------------------------------------------------

## 4️⃣ Performance Engineering

原则：

> 只优化最慢的20%。

建立性能直觉。

------------------------------------------------------------------------

## 5️⃣ Rewrite It

自己实现： - mini-redis - mini-mysql - mini-mq

第一性原理学习。

------------------------------------------------------------------------

# 五、项目序列设计

## Phase 1 --- Storage Foundation

### mini-mysql

实现： - B+Tree - WAL - Buffer Cache - 简单事务

------------------------------------------------------------------------

## Phase 2 --- Cache Engineering

### mini-redis

版本递进：

-   V0：TCP + KV\
-   V1：Pipeline + TTL\
-   V2：AOF + Snapshot\
-   V3：Replication\
-   V4：Cluster + Consistent Hash

------------------------------------------------------------------------

## Phase 3 --- Log-based System

### mini-mq

实现： - partition - segment file - consumer offset

理解顺序写带来的吞吐优势。

------------------------------------------------------------------------

## Phase 4 --- RPC Framework

实现： - 服务发现 - retry - timeout - 负载均衡

理解网络是系统边界。

------------------------------------------------------------------------

## Phase 5 --- Coordination

实现简化版 Raft，理解： - CAP - 共识算法 - Leader election

------------------------------------------------------------------------

## Phase 6 --- Gateway

能力： - 限流 - 熔断 - routing - auth

------------------------------------------------------------------------

# 六、Chaos Engineering

主动制造故障：

-   注入延迟
-   丢包
-   kill node
-   磁盘变慢

训练系统恢复能力。

# 十、关键认知原则

## 实现 : 源码 = 7 : 3

优先自己实现。

## 按场景学习，而不是按技术学习

例如：

-   秒杀 → Redis + MQ + 限流\
-   MySQL瓶颈 → 分库分表 + Cache

这是架构驱动学习。
